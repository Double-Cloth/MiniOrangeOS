# 自主实现与来源登记

> 状态：前置模板。后续每个任务完成后必须更新，不得等答辩前集中补。

## 原则

MiniOrangeOS 必须证明核心代码是自主实现。允许参考公开文档、CPU 手册、BIOS 资料、工具手册和教材概念；禁止复制教学操作系统源码后改名。

Codex 生成代码也必须被开发者审查和解释。答辩时需要说明：

- 参考了什么；
- 哪些代码由项目实现；
- 哪些行为通过测试证明；
- 哪些限制是有意取舍。

## 禁止计入自主核心代码

- 编译产物；
- 磁盘镜像；
- 自动生成表格；
- 第三方库；
- 复制的头文件；
- 大量重复测试数据；
- 由工具生成且未经审查的代码。

## 代码量报告要求

后续 `make loc` 必须至少区分：

- Boot/Loader 汇编；
- 内核 C；
- 内核汇编；
- 用户程序；
- 用户 libc；
- 工具；
- 测试；
- 文档；
- 自动生成文件；
- 第三方文件。
